# 抖音关键词视频采集功能设计文档

**日期**: 2026-04-01
**功能名称**: 抖音关键词视频采集 (dy_video_list_crawler)

## 1. 功能概述

根据关键词搜索抖音视频，导出视频列表和达人数据到 Excel 文件。用户可按需选择数据类型，减少不必要的页面访问。

## 2. 输入参数

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| 搜索关键字 | 文本 | 搜索关键词 | 必填 |
| 视频类型 | 选择 | 全部/视频/图文 | 全部 |
| 搜索规则 | 选择 | 综合/最多点赞/最新发布 | 综合 |
| 视频总数 | 数字 | 爬取视频数量上限 | 20 |
| 数据类型 | 多选 | 视频列表/视频数据/达人数据 | 全选 |

## 3. 数据类型定义

| 选项 | 包含字段 | 数据来源 |
|------|----------|----------|
| **视频列表** | 视频标题、视频链接、达人昵称、达人主页链接 | 搜索页直接获取 |
| **视频数据** | 点赞数、评论数、分享数、收藏数 | 进视频详情页获取 |
| **达人数据** | 粉丝数、获赞数、作品数 | 进达人主页获取 |

## 4. 爬取流程

```
1. 打开抖音搜索页面（关键词 + 筛选条件）
   ↓
2. 滚动页面，收集视频列表
   - 视频标题、视频链接
   - 达人昵称、达人主页链接（搜索页可见）
   ↓
3. 根据用户选择决定后续操作：
   ├─ 若选了「视频数据」→ 逐个进详情页获取互动数据
   ├─ 若选了「达人数据」→ 去重达人 → 批量进主页获取基础数据
   └─ 若都没选 → 直接导出
   ↓
4. 合并数据，导出 Excel
```

## 9. 技术实现要点

### 9.1 搜索页数据提取

使用 DOM 选择器提取：
- 视频链接：`a[href*="/video/"]`
- 视频标题：视频卡片标题元素
- 达人昵称：视频卡片作者元素
- 达人主页链接：作者头像/昵称的链接

### 9.2 视频详情页数据提取

进入视频详情页后提取：
- 点赞数、评论数、分享数、收藏数：视频信息栏元素

### 9.3 达人主页数据提取

进入达人主页后提取：
- 粉丝数、获赞数、作品数：主页数据栏元素

### 9.4 滚动加载

参考 `douyin_comment_crawler/demo.js` 中的 `crawler_videos` 函数：
- 滚动页面触发懒加载
- 检测是否到达底部
- 去重视频链接
- 达到数量上限后停止

### 9.5 达人去重

使用 `Set` 或对象存储已访问的达人主页链接，避免重复访问。

## 10. 错误处理

- 关键词为空：提示用户输入
- 页面加载失败：重试机制（最多 3 次）
- 数据提取失败：跳过该条目，记录日志
- 导出失败：提示用户并保留内存数据

## 11. 额外功能（可选）

- **进度显示**：爬取过程中显示当前进度（第 X/Y 个视频）
- **暂停/继续**：支持中途暂停和继续爬取
- **预览功能**：爬取完成后预览数据，确认后再导出
